架构总览

  整个"记忆面包"由 3 种语言 + 7 个核心组件构成：

  感知（Rust 引擎）→ 理解（Python AI Sidecar）→ 执行（Rust 执行器）→ 展示（Tauri/React UI）

  ---
  7 个组件 & 语言分工

  ┌─────┬─────────────────────┬──────────────────────────┬───────────────────────────────────────────────┐
  │  #  │        组件         │           语言           │                   核心职责                    │
  ├─────┼─────────────────────┼──────────────────────────┼───────────────────────────────────────────────┤
  │ 1   │ 核心采集引擎        │ Rust                     │ 事件监听、Accessibility Tree 抓取、截图       │
  ├─────┼─────────────────────┼──────────────────────────┼───────────────────────────────────────────────┤
  │ 2   │ AI 推理 Sidecar     │ Python                   │ OCR、ASR（Whisper）、VLM、Embedding、PII 脱敏 │
  ├─────┼─────────────────────┼──────────────────────────┼───────────────────────────────────────────────┤
  │ 3   │ 本地存储层          │ SQLite                   │ 时间线存储（FTS5 全文检索）+ Qdrant 向量库    │
  ├─────┼─────────────────────┼──────────────────────────┼───────────────────────────────────────────────┤
  │ 4   │ 桌面 UI             │ TypeScript/React + Tauri │ 悬浮窗、问答面板、接管确认弹窗                │

  ---
  建议的代码生成顺序

  从底层到上层，依次生成：

  1. db-schema → 2. ipc-protocol → 3. storage → 4. capture → 5. ocr/embedding/rag → 6. api-server → 7. executor → 8. desktop-ui → 9. asr/vlm（可选）

  你在用 Claude Code 生成各组件时，可以把对应的组件章节内容一并提供给它作为上下文，生成的代码会更准确。
