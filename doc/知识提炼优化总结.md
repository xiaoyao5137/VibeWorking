### 2. 知识库推理内容过于精简

**问题**：
- 当前只生成 50-200 字的简短摘要
- 缺少详细内容，后期追溯困难
- 无法区分"概述"和"明细"

**解决方案**：
- 修改数据结构，增加 `overview` 和 `details` 字段
- `overview`：简洁描述正在做什么（30-100字）
- `details`：详细记录具体内容（200-500字）
- 保留 `summary` 字段用于向后兼容

**数据库迁移**：
- 创建迁移脚本 `004_add_overview_details.sql`
- 添加 `overview` 和 `details` 字段
- 更新 FTS 全文索引
- 重建触发器

**修改文件**：
- `ai-sidecar/knowledge/extractor_v2.py` - 更新 SYSTEM_PROMPT
- `ai-sidecar/knowledge/manager.py` - 更新 `add_entry` 方法
- `ai-sidecar/background_processor.py` - 更新数据库插入逻辑
- `ai-sidecar/migrations/004_add_overview_details.sql` - 数据库迁移

**LLM 配置调整**：
- 增加 `num_predict` 从 256 到 1024 tokens
- 支持生成更详细的内容

### 3. 相同事情应该合并到同一条知识

**问题**：
- 前后时刻采集到的相同工作内容被分散存储
- 只是简单计数，没有合并详细内容

**解决方案**：
- 修改知识去重逻辑
- 发现相似知识时，合并 `details` 内容而不是简单计数
- 追加新的细节信息，标注时间戳
- 使用语义相似度（余弦相似度 >= 0.85）判断是否为同一事情

**实现细节**：
```python
# 合并明细：保留原有内容，追加新内容
merged_details = existing_details
if details and details not in existing_details:
    merged_details += f"\n\n--- 补充 ({datetime.now().strftime('%Y-%m-%d %H:%M')}) ---\n{details}"
```

**修改文件**：
- `ai-sidecar/knowledge/extractor_v2.py` - 更新 `extract_sync` 方法

---

### 4. 向量计算错误修复

**问题**：
- `unsupported operand type(s) for *: 'EmbeddingVector' and 'EmbeddingVector'`
- 向量模型返回的是 `EmbeddingVector` 对象，不是 numpy 数组

**解决方案**：
- 修改 `_find_similar_knowledge` 方法
- 正确提取 `EmbeddingVector.vector` 属性
- 转换为 numpy 数组进行计算

**修改代码**：
```python
new_embedding = self.embedding_model.encode([overview])[0]
new_vector = np.array(new_embedding.vector)
```

**修改文件**：
- `ai-sidecar/knowledge/extractor_v2.py` - 更新 `_find_similar_knowledge` 方法

---

### 5. JSON 解析错误优化

**问题**：
- LLM 生成的 JSON 包含未转义的换行符
- 导致 JSON 解析失败

**解决方案**：
- 优化 SYSTEM_PROMPT，明确要求：
  - 不要在字符串中使用换行符
  - 使用空格代替换行符
  - 确保输出有效的 JSON 格式
  - 转义特殊字符

**修改文件**：
- `ai-sidecar/knowledge/extractor_v2.py` - 更新 SYSTEM_PROMPT
