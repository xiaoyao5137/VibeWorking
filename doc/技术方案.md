> 版本：v1.0 | 日期：2026-03-03

## 一、产品定位与核心目标

"记忆面包"是一款运行在本地的 AI 自动化办公助手，其核心能力链路为：

```
屏幕感知（Perception） → 意图理解（Cognition） → 任务执行（Action）
```

三大差异化能力：
1. **无感采集**：事件驱动 + Accessibility Tree 优先，CPU 占用 < 10%
2. **本地私有**：数据 100% 本地存储，可选端侧大模型
3. **按键精灵式接管**：不只"给建议"，直接替用户操控键鼠完成任务

## 三、技术组件详细说明

### 组件 1：核心采集引擎（Core Capture Engine）

| 项目 | 技术选型 |
|------|---------|
| **编程语言** | **Rust** |
| 跨平台框架 | 原生 Rust（不依赖额外框架） |
| macOS 事件监听 | `core-graphics` crate + `CGEvent` API |
| Windows 事件监听 | `windows` crate + `SetWindowsHookEx` |
| macOS Accessibility | `accessibility` crate（封装 AXUIElement API） |
| Windows Accessibility | `uiautomation` crate（封装 UIAutomation COM） |
| 截图捕获 | `screenshots` crate（跨平台，硬件加速） |
| 图像压缩 | `image` crate（JPEG 80% 质量压缩） |
| 数据库写入 | `rusqlite`（SQLite 绑定） |
| 事件队列 | `tokio` 异步运行时 + `mpsc` channel |

**核心逻辑：**
```
全局事件钩子 ──触发──> 变化检测器 ──判断──> 是否实质变化？
                                              │ Yes
                                        截图 + Accessibility Tree 抓取
                                              │
                                        写入 SQLite + 发送给 AI Sidecar
```

**关键策略：**
- Accessibility Tree 优先，失败才降级 OCR
- 事件去抖（debounce）：键盘输入停顿 2 秒才触发
- 隐私窗口过滤：检测到密码框 / 隐身模式浏览器自动跳过

## 六、本地知识库生成与管理

### 6.1 知识提炼架构

```
OCR 原始文本 ──> 本地小模型提炼 ──> 结构化知识条目 ──> 向量化存储 ──> RAG 检索
     │                │                    │                │              │
  (噪声多)      (去噪+摘要)         (可验证)         (Qdrant)      (用户查询)
```

**核心理念**：
- OCR 采集的原始文本包含大量噪声（UI 元素、重复内容、无意义片段）
- 使用本地小参数量 LLM 进行智能提炼，生成高质量知识条目
- 用户可在 UI 中查看、编辑、删除知识条目，形成人机协同的知识库

### 6.2 本地 LLM 模型选型

#### 推荐模型：Qwen3.5-4B

| 项目 | 参数 |
|------|------|
| **模型名称** | Qwen3.5-4B (Small Series) |
| **参数量** | 4B（40 亿） |
| **量化版本** | Q4_K_M（约 2.5GB 显存） |
| **推理速度** | ~25 tokens/s（M1 Pro / RTX 3060） |
| **上下文长度** | 128K tokens |
| **中文能力** | ⭐⭐⭐⭐⭐（原生中文训练） |
| **指令遵循** | ⭐⭐⭐⭐⭐（原生多模态） |
| **部署方式** | Ollama 一键安装 |
| **发布时间** | 2026 年 2 月 |

**选择理由**：
1. **参数量适中**：4B 模型在 8GB 内存设备上流畅运行
2. **中文优秀**：Qwen3.5 系列对中文支持最好，适合办公场景
3. **原生多模态**：支持视觉理解，可处理截图内容
4. **超长上下文**：128K tokens，可处理大量采集记录
5. **Ollama 支持**：一行命令即可部署：`ollama pull qwen3.5:4b`
6. **性能提升**：比 Qwen2.5 提升 30%+ 推理速度

**备选方案**：
- **Qwen3.5-2B**：更轻量（2B），适合低配设备
- **Qwen3.5-0.8B**：超轻量（0.8B），移动端部署
- **Qwen3.5-9B**：更强性能（9B），高配设备推荐

**参考资料**：
- [Qwen3.5 Small Models Guide](https://apidog.com/blog/use-qwen-3-5-small-models/)
- [Qwen3.5 Ollama 使用指南](https://apidog.com/blog/use-qwen-3-5-with-ollama/)
- [Qwen3.5 性能评测](https://artificialanalysis.ai/articles/qwen3-5-small-models)

### 6.3 知识提炼 Prompt 设计

#### 系统 Prompt（固定）

```
你是一个专业的工作记录提炼助手。你的任务是从 OCR 识别的屏幕文本中提取有价值的工作信息。

**提炼规则**：
1. 忽略 UI 元素（按钮、菜单、状态栏等）
2. 提取核心工作内容（会议记录、文档内容、代码片段、聊天记录等）
3. 生成简洁的摘要（50-200 字）

}
```

### 6.5 知识提炼流程

```python
# ai-sidecar/knowledge/extractor.py

from ollama import Client
import json

class KnowledgeExtractor:
    def __init__(self):
        self.client = Client()
        self.model = "qwen3.5:4b"

    async def extract(self, capture_data: dict) -> dict:
        """从采集数据中提炼知识"""

        # 1. 构建 prompt
        prompt = self._build_prompt(capture_data)

        # 2. 调用本地 LLM
        response = self.client.chat(
            model=self.model,
            messages=[
                {"role": "system", "content": SYSTEM_PROMPT},
                {"role": "user", "content": prompt}
            ],
            format="json",  # 强制 JSON 输出
            options={
                "temperature": 0.3,  # 降低随机性
                "num_predict": 256,  # 限制输出长度
            }
        )

        # 3. 解析结果
        result = json.loads(response['message']['content'])

        # 4. 跳过无价值内容
        if result.get('summary') == 'SKIP':
            return None

        # 5. 返回结构化知识
        return {
            'capture_id': capture_data['id'],
            'summary': result['summary'],
            'entities': json.dumps(result.get('entities', [])),
            'category': result.get('category', '其他'),
            'importance': result.get('importance', 3),
        }
```

  }

### 6.8 知识库性能优化

| 优化项 | 策略 |
|--------|------|
| **批量提炼** | 每 10 条采集记录批量调用 LLM，减少推理开销 |
| **异步队列** | 使用 Celery / RQ 异步处理，不阻塞采集主流程 |
| **缓存机制** | 相似内容（余弦相似度 > 0.95）跳过重复提炼 |
| **增量更新** | 只对新采集的数据进行提炼，已有知识不重复处理 |
| **用户反馈** | 用户编辑/删除的条目作为负样本，持续优化模型 |

## 七、关键技术风险与应对

| 风险点 | 描述 | 应对方案 |
|--------|------|---------|
| **Accessibility 权限** | macOS/Windows 需用户手动授权辅助功能 | 首次启动引导页 + 权限检测重试机制 |
| **OCR 中文准确率** | 复杂排版下 OCR 失准 | PaddleOCR v3 + 后处理纠错规则 |
| **本地 VLM 性能** | 端侧 VLM 推理延迟高 | 异步队列处理，不阻塞采集主流程 |
| **隐私边界判断** | 自动跳过密码框的误判/漏判 | 多重规则：AX role=password + 黑名单应用列表 |
| **键鼠接管误触** | 全自动模式下 AI 操作失误 | 默认半自动（用户确认），全自动需显式开启 |
| **SQLite 并发写入** | 多线程同时写入导致锁冲突 | WAL 模式 + 单写线程（tokio channel 串行化） |
| **跨分辨率兼容** | 截图坐标在高 DPI 屏幕失准 | 统一使用逻辑坐标，Accessibility API 天然兼容 |
