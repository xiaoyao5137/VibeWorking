## 核心设计

### 知识提炼流程
```
OCR 原始文本 → 本地 LLM 提炼 → 结构化知识 → 向量化存储 → UI 可视化验证
```

## 本地 LLM 模型选型

### 推荐方案：Qwen2.5-3B-Instruct

| 项目 | 参数 |
|------|------|
| 参数量 | 3B（30 亿） |
| 量化版本 | Q4_K_M（约 2GB 显存） |
| 推理速度 | ~20 tokens/s（M1 Pro / RTX 3060） |
| 上下文长度 | 32K tokens |
| 中文能力 | ⭐⭐⭐⭐⭐ |
| 部署方式 | `ollama pull qwen2.5:3b-instruct` |

### 选择理由

1. **参数量适中**：3B 模型在 8GB 内存设备上流畅运行
2. **中文优秀**：Qwen 系列对中文支持最好，适合办公场景
3. **指令遵循强**：Instruct 版本能准确理解提炼任务
4. **社区活跃**：Hugging Face 上有丰富的微调资源
5. **一键部署**：Ollama 支持，无需复杂配置

### 备选方案

- **Qwen2.5-1.5B-Instruct**：更轻量（1.5B），适合低配设备
- **Phi-3-mini-4k-instruct**：微软出品（3.8B），英文场景更优
- **Gemma-2-2B-it**：Google 出品（2B），多语言均衡

## 微调流程

### 使用 LLaMA-Factory

```bash
# 1. 安装
git clone https://github.com/hiyouga/LLaMA-Factory.git
cd LLaMA-Factory
pip install -e .

# 2. 准备数据集
# 将自建 SOP 数据放到 data/sop_dataset.json

# 3. 启动微调
llamafactory-cli train examples/train_qlora/qwen2_lora_sft.yaml

# 4. 合并权重
llamafactory-cli export examples/merge_lora/qwen2_lora_sft.yaml

# 5. 部署到 Ollama
ollama create memory-bread-qwen2.5:3b -f Modelfile
```

### 微调参数建议

- **方法**：LoRA（低秩适应）
- **Rank**：8-16
- **学习率**：5e-5
- **Batch Size**：4-8
- **Epochs**：3-5
- **训练时间**：1-2 小时（RTX 3060）

## 知识库数据结构

### SQLite 表结构

```sql
CREATE TABLE knowledge_entries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    capture_id INTEGER NOT NULL,           -- 关联原始采集记录
    summary TEXT NOT NULL,                 -- 提炼后的摘要
    entities TEXT,                         -- JSON 数组：["实体1", "实体2"]
    category TEXT,                         -- 分类：会议/文档/代码/聊天/其他
    importance INTEGER DEFAULT 3,          -- 重要性：1-5
    user_verified BOOLEAN DEFAULT 0,       -- 用户是否验证过
    user_edited BOOLEAN DEFAULT 0,         -- 用户是否编辑过
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (capture_id) REFERENCES captures(id)
);

-- 全文搜索索引
CREATE VIRTUAL TABLE knowledge_fts USING fts5(
    summary,
    entities,
    content='knowledge_entries',
    content_rowid='id'
);
```

## UI 知识库管理界面

### 功能设计

```
┌─────────────────────────────────────────────────────────────┐
│  知识库管理                                    [刷新] [设置]  │
├─────────────────────────────────────────────────────────────┤
│  筛选：[全部▼] [会议] [文档] [代码] [聊天]    搜索：[____]  │
├─────────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────────┐  │
│  │ 📝 飞书会议：Q1 产品规划讨论           ⭐⭐⭐⭐⭐      │  │
│  │ 2026-03-05 10:30  |  会议  |  已验证                   │  │
│  │ 摘要：讨论了 Q1 产品路线图，重点是 AI 功能集成...     │  │
│  │ 实体：张三、李四、记忆面包、Q1                       │  │
│  │ [查看原文] [编辑] [删除]                              │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 核心功能

1. **查看知识条目**：列表展示所有提炼的知识
2. **筛选分类**：按会议/文档/代码/聊天分类
3. **搜索**：全文搜索 + 向量语义搜索
4. **验证**：用户确认提炼是否正确
5. **编辑**：修改摘要、实体、分类
6. **删除**：移除错误的知识条目
7. **查看原文**：跳转到原始采集记录

## 性能指标

| 指标 | 目标值 |
|------|--------|
| 知识提炼延迟 | < 2s/条（本地 3B 模型） |
| 知识库查询延迟 | < 300ms（向量检索 + 全文搜索） |
| 批量提炼速度 | 10 条/次，减少推理开销 |
| 内存占用 | < 2GB（包含模型） |

## 优化策略

| 优化项 | 策略 |
|--------|------|
| **批量提炼** | 每 10 条采集记录批量调用 LLM |
| **异步队列** | 使用 Celery / RQ 异步处理 |
| **缓存机制** | 相似内容（余弦相似度 > 0.95）跳过重复提炼 |
| **增量更新** | 只对新采集的数据进行提炼 |
| **用户反馈** | 用户编辑/删除的条目作为负样本 |
