## 一、项目概览

### 1.1 产品定位

记忆面包（记忆面包）是一款**本地化 AI 自动化办公助手**，核心能力链路为：

```
屏幕感知（Perception） → 意图理解（Cognition） → 任务执行（Action）
```

三大差异化能力：
- **无感采集**：事件驱动 + Accessibility Tree 优先，CPU 占用 < 10%
- **本地私有**：数据 100% 本地存储，可选端侧大模型
- **按键精灵式接管**：不只"给建议"，直接替用户操控键鼠完成任务

### 1.2 技术架构


### 2.2 AI Sidecar（Python）

#### 2.2.1 dispatcher.py - IPC 任务分发器

**核心职责**：
- 接收来自 Rust Core Engine 的 IPC 请求
- 根据 `task.type` 路由到对应 Worker
- 懒加载 AI 模型（首次使用时初始化）

**关键代码**：
```python
class Dispatcher:
    async def dispatch(self, req: IpcRequest) -> IpcResponse:
        if req.task.type == "ping":
            return IpcResponse.make_ok(req.id, PingResult())
        elif req.task.type == "ocr":
            worker = self._get_ocr_worker()
            return await worker.handle(req)
        elif req.task.type == "embed":
            worker = self._get_embed_worker()
            return await worker.handle(req)
        # ...
```

懒加载策略：
- 首次调用时才加载模型（避免启动时内存占用过高）
- 模型加载失败时返回错误响应，不阻塞其他任务

## 三、数据流解析

### 3.1 采集流程

```
用户操作（点击/输入/切换应用）
    ↓
系统事件钩子捕获
    ↓
CaptureEngine 接收事件
    ↓
PrivacyFilter 隐私检查
    ↓ (通过)
截图 + Accessibility Tree 抓取
    ↓
写入 SQLite (captures 表)
    ↓
发送 IPC 请求到 AI Sidecar
    ↓
OCR Worker 识别文字
    ↓
更新 captures.ocr_text
    ↓
Embedding Worker 生成向量
    ↓
写入 Qdrant + vector_index 表
```

### 3.2 查询流程

```
用户在 UI 输入查询
    ↓
发送 HTTP POST /api/captures/search
    ↓
API Server 接收请求
    ↓
CaptureRepo.search_fts() 全文搜索
    ↓
返回匹配的 Capture 记录
    ↓
UI 展示时间线 + 截图预览
```

### 3.3 执行流程

```
用户选择"自动执行"
    ↓
LLM 生成操作序列 (Action[])
    ↓
发送 HTTP POST /api/actions/execute
    ↓
Executor 逐步执行
    ↓
记录到 action_logs 表
    ↓
返回执行结果
```

## 四、关键技术点

### 4.1 性能优化

1. **事件驱动采集**
   - 不使用轮询，仅在实质变化时触发
   - CPU 占用 < 10%

2. **Accessibility Tree 优先**
   - 比截图 OCR 快 10 倍
   - 仅在 AX 信息不足时才截图

3. **懒加载 AI 模型**
   - 首次使用时才加载
   - 避免启动时内存占用过高

4. **SQLite WAL 模式**
   - 支持并发读
   - 写入不阻塞查询

5. **JPEG 压缩**
   - 80% 质量，单张 < 200KB
   - 单日数据增量 60-100MB

### 4.2 隐私保护

1. **本地存储**
   - 所有数据 100% 本地
   - 不上传云端

2. **隐私过滤**
   - 自动跳过密码框
   - 黑名单应用（1Password、钉钉）
   - URL 模式匹配（银行、支付）

3. **PII 脱敏**
   - 使用 Presidio 检测敏感信息
   - 调用云端 LLM 前先脱敏

4. **用户控制**
   - 可手动暂停采集
   - 可删除历史记录
   - 可设置数据保留期限

### 4.3 可靠性保障

1. **错误处理**
   - 所有 Result 类型显式处理
   - 统一错误类型和日志

2. **降级策略**
   - AI Sidecar 启动失败时，仅保留 AX 采集
   - OCR 失败时，使用 AX 文本

3. **数据清理**
   - 自动清理过期截图
   - VACUUM 回收磁盘空间

4. **测试覆盖**
   - capture 模块：66/66 tests
   - ocr 模块：59/59 tests
   - 单元测试 + 集成测试

## 七、代码规范

### 7.1 Rust 代码规范

1. **错误处理**
   - 使用 `Result<T, E>` 而非 `panic!`
   - 自定义错误类型实现 `std::error::Error`

2. **异步编程**
   - 使用 `tokio` 运行时
   - I/O 操作必须 `async`

3. **命名规范**
   - 模块名：`snake_case`
