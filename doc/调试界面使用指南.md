# WorkBuddy 闲时计算调试界面使用指南

## 功能概述

调试界面提供以下功能：

1. **系统状态监控**
   - CPU/内存使用率实时监控
   - 闲时检测器状态
   - 模型加载状态
   - 任务队列统计

2. **任务测试**
   - 一键提交各类型任务
   - 查看任务执行结果
   - 监控任务队列

3. **模型管理**
   - 查看已加载模型
   - 手动卸载模型
   - 监控内存占用

4. **闲时控制**
   - 手动触发闲时任务
   - 查看下次执行时间
   - 调整执行参数

---

## 启动方法

### 1. 安装依赖

```bash
pip install fastapi uvicorn websockets
```

### 2. 启动调试界面

```bash
cd ai-sidecar
python start_debug_ui.py
```

### 3. 访问界面

在浏览器打开：
```
http://localhost:8080
```

## 使用场景

### 场景 1: 测试 OCR 实时处理

1. 点击 "OCR 任务" 的 [提交任务] 按钮
2. 观察右侧日志：
   ```
   10:30:00 - 提交 OCR 任务: ocr-test-1234567890
   10:30:00 - OCR 任务已提交（实时处理）
   10:30:01 - OCR 任务完成 (耗时 0.8 秒)
   ```
3. 确认模型状态：PaddleOCR 保持常驻

### 场景 2: 测试 Embedding 闲时处理

1. 点击 "Embedding 任务" 的 [提交任务] 按钮 10 次
2. 观察任务统计：
   ```
   待处理: 10
     - 闲时任务: 10
   ```
3. 等待闲时触发（或点击 [手动触发闲时任务]）
4. 观察日志：
   ```
   10:31:00 - 进入闲时模式
   10:31:00 - 开始处理 10 个 Embedding 任务
   10:31:00 - 开始加载模型: BGE-M3-INT8
   10:31:03 - 模型加载完成
   10:31:15 - Embedding 任务处理完成: 10 个
   10:31:15 - 卸载模型: BGE-M3-INT8
   ```
5. 确认模型状态：BGE-M3 已卸载

### 场景 3: 测试 LLM 按需加载

1. 点击 "LLM 任务" 的 [提交任务] 按钮
2. 观察模型状态：
   ```
   ✓ Qwen2.5-3B (2500MB) - 已加载
   总内存占用: 2550 MB
   ```
3. 等待 2 分钟
4. 观察模型状态：
   ```
   ✗ Qwen2.5-3B (2500MB) - 未加载
   总内存占用: 50 MB
   ```

### 场景 4: 监控内存占用

1. 提交多个不同类型的任务
2. 观察 "模型状态" 面板的内存占用变化
3. 确认同时最多加载 2 个模型
4. 点击 [卸载所有模型] 测试强制卸载

### 场景 5: 调试闲时检测

1. 观察 "闲时检测器" 面板
2. 查看当前 CPU/内存是否满足条件
3. 查看稳定进度条
4. 如果长时间不触发，点击 [手动触发闲时任务]

## API 端点

### 获取系统状态
```bash
curl http://localhost:8080/api/status
```

### 提交任务
```bash
# OCR 任务
curl -X POST http://localhost:8080/api/task/ocr

# Embedding 任务
curl -X POST http://localhost:8080/api/task/embedding

# VLM 任务
curl -X POST http://localhost:8080/api/task/vlm

# ASR 任务
curl -X POST http://localhost:8080/api/task/asr

# LLM 任务
curl -X POST http://localhost:8080/api/task/llm
```

```bash
curl -X POST http://localhost:8080/api/trigger_idle
```

### 卸载所有模型
```bash
curl -X POST http://localhost:8080/api/model/unload_all
```

## 故障排查

### 问题 1: 界面无法访问

**检查**:
```bash
# 确认服务已启动
ps aux | grep start_debug_ui

# 确认端口未被占用
lsof -i :8080
```

**解决**:
```bash
# 更换端口
# 修改 start_debug_ui.py 中的 port=8080
```

### 问题 2: 闲时任务从不执行

**检查**:
- 查看 "闲时检测器" 面板
- 确认 CPU/内存是否满足条件
- 确认是否在充电

**解决**:
- 点击 [手动触发闲时任务]
- 或修改 dispatcher_v2.py 中的阈值

### 问题 3: 模型未正确卸载

**检查**:
- 查看 "模型状态" 面板
- 确认已加载模型数量

**解决**:
- 点击 [卸载所有模型]
- 查看日志确认卸载成功

## 开发说明

### 添加新的任务类型

1. 在 `debug_ui.py` 中添加 API 端点:
```python
@app.post("/api/task/new_task")
async def submit_new_task():
    task = Task(
        task_id=f"new-test-{datetime.now().timestamp()}",
        task_type=TaskType.NEW_TASK,
        priority=TaskPriority.IDLE_HIGH,
        payload={"data": "test"},
    )
    success = dispatcher._task_scheduler.submit_task(task)
    return {"success": success, "task_id": task.task_id}
```

2. 在 HTML 模板中添加按钮:
```html
<div class="task-card">
    <h3>新任务</h3>
    <p>类型: 闲时队列</p>
    <button onclick="submitTask('new_task')">提交任务</button>
</div>
```

### 自定义监控指标

修改 `get_status()` 函数添加新的监控数据:
```python
@app.get("/api/status")
async def get_status():
    # ... 现有代码 ...

    return {
        # ... 现有数据 ...
        "custom_metrics": {
            "your_metric": your_value,
        },
    }
```
