### 2. 实现向量 RAG 查询功能

**架构说明**：
记忆面包 使用 **Tauri** 框架：
- 前端：React + TypeScript
- 渲染：系统 WebView (macOS 上是 WKWebView)
- 后端：Rust (core-engine) + Python (ai-sidecar)
- 不是纯原生组件，也不是传统的内嵌 H5

**实现方案**：

#### 2.1 创建 RAG HTTP 服务 (ai-sidecar)

**新增文件**：
- `ai-sidecar/rag_api_server.py` - RAG 查询 HTTP 服务（端口 7071）
- `ai-sidecar/rag/worker.py` - RAG Worker（IPC 任务处理器）

**功能**：
- 提供 `/query` API 端点
- 集成完整的 RAG pipeline：
  - 向量检索（Qdrant）
  - 关键词检索（FTS5）
  - RRF 合并
  - LLM 推理（Ollama）

#### 2.2 更新 Core Engine

**修改文件**：
- `core-engine/src/api/handlers/query.rs` - 更新查询逻辑
- `core-engine/Cargo.toml` - 添加 reqwest 依赖

**新逻辑**：
1. 优先调用 RAG 服务（端口 7071）进行智能问答
2. 如果 RAG 服务不可用，降级为关键词搜索
3. 支持超时和错误处理

#### 2.3 更新 IPC 协议

**修改文件**：
- `shared/ipc-protocol/python/memory_bread_ipc/message.py`
- `ai-sidecar/dispatcher.py`

**新增消息类型**：
- `RagRequest` - RAG 查询请求
- `RagResult` - RAG 查询结果
- 在 dispatcher 中添加 RAG worker 支持

## 📊 技术细节

### RAG Pipeline 流程

```
用户问题
    ↓
Core Engine (/query)
    ↓
HTTP 调用 RAG 服务 (7071)
    ↓
RAG Pipeline
    ├─ Query Embedding (BGE-M3)
    ├─ 向量检索 (Qdrant)
    ├─ 关键词检索 (FTS5)
    ├─ RRF 合并
    ├─ Prompt 组装
    └─ LLM 推理 (Ollama)
    ↓
返回智能回答 + 上下文
    ↓
Desktop UI 显示
```

### 降级策略

如果 RAG 服务不可用（未启动或超时），Core Engine 会自动降级为：
1. 使用 SQL LIKE 进行关键词匹配
2. 从知识库和采集记录中检索
3. 返回简单的摘要（不调用 LLM）
