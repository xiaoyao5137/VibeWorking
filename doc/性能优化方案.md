# 记忆面包 系统卡死问题诊断与解决方案

## 问题诊断

### 🔴 高危问题

#### 1. **无限制的截图循环（CPU/内存杀手）**
**位置**: `core-engine/src/capture/listener.rs:39`

```rust
loop {
    ticker.tick().await;
    debug!("触发定时采集事件");
    if tx.send(CaptureEvent::Periodic).await.is_err() {
        info!("采集引擎已关闭，停止监听器");
        break;
    }
}
```

**问题**:
- 默认每 30 秒触发一次全屏截图 + OCR + Embedding
- 每次截图调用 `xcap::Monitor::all()` 获取所有显示器
- JPEG 编码（CPU 密集）+ 磁盘 I/O（每次 60-100KB）
- 如果 OCR/Embedding 处理慢，会导致任务堆积

**影响**: CPU 持续高负载，内存泄漏风险

---

#### 2. **osascript 阻塞调用（macOS 特有）**
**位置**: `core-engine/src/capture/ax.rs:67-71`

```rust
let output = Command::new("osascript")
    .arg("-e")
    .arg(basic_script)
    .output()
    .ok()?;
```

**问题**:
- 每次采集都同步调用 `osascript`（2 次调用）
- 如果系统权限弹窗未授予，会一直阻塞
- 在 30 秒循环中，可能导致主线程卡死

**影响**: UI 冻结，系统响应缓慢

---

#### 3. **SQLite 写入锁竞争**
**位置**: `core-engine/src/storage/db.rs`

```rust
pub struct StorageManager {
    conn: Arc<Mutex<Connection>>,
}
```

**问题**:
- 所有写入操作共享一个 `Mutex<Connection>`
- 高频采集（30 秒/次）+ OCR 回调写入 = 锁竞争
- SQLite WAL 模式下，`VACUUM` 操作会阻塞所有读写

**影响**: 数据库死锁，内存占用飙升

---

#### 4. **IPC 超时设置过长**
**位置**: `core-engine/src/ipc.rs:87`

```rust
timeout: Duration::from_secs(120), // 120 秒超时
```

**问题**:
- 如果 AI Sidecar 未启动或崩溃，每次 OCR 调用会阻塞 120 秒
- 在 30 秒采集循环中，会导致任务队列爆满

**影响**: 内存泄漏，系统假死

---

### 🟡 中危问题

#### 5. **无采集频率自适应**
- 无论用户是否在使用电脑，都以固定频率采集
- 没有"空闲检测"机制（如 5 分钟无操作则暂停）

#### 6. **缺少资源限制**
- 没有最大截图数量限制
- 没有磁盘空间检查（可能填满硬盘）
- 没有内存使用监控

#### 修复 1: 添加采集开关和频率控制

**文件**: `core-engine/src/capture/listener.rs`

```rust
use std::sync::atomic::{AtomicBool, Ordering};
use std::sync::Arc;

pub struct ListenerConfig {
    pub interval_secs: u64,
    pub enabled: Arc<AtomicBool>,  // 新增：运行时开关
    pub idle_threshold_secs: u64,  // 新增：空闲阈值（默认 300 秒）
}

impl Default for ListenerConfig {
    fn default() -> Self {
        Self {
            interval_secs: 60,  // 改为 60 秒（降低频率）
            enabled: Arc::new(AtomicBool::new(true)),
            idle_threshold_secs: 300,
        }
    }
}

pub async fn start_listener(
    config: ListenerConfig,
    tx: mpsc::Sender<CaptureEvent>,
) {
    let mut ticker = interval(Duration::from_secs(config.interval_secs));
    let mut last_activity = SystemTime::now();

    loop {
        ticker.tick().await;

        // 检查是否启用
        if !config.enabled.load(Ordering::Relaxed) {
            tokio::time::sleep(Duration::from_secs(5)).await;
            continue;
        }

        // 检查空闲时间
        if let Ok(idle_secs) = get_system_idle_time() {
            if idle_secs > config.idle_threshold_secs {
                debug!("系统空闲 {} 秒，跳过采集", idle_secs);
                continue;
            }
        }

        // 发送采集事件（带超时保护）
        match tokio::time::timeout(
            Duration::from_secs(5),
            tx.send(CaptureEvent::Periodic)
        ).await {
            Ok(Ok(_)) => {},
            Ok(Err(_)) => {
                info!("采集引擎已关闭");
                break;
            }
            Err(_) => {
                warn!("发送采集事件超时，跳过本次");
            }
        }
    }
}

// 获取系统空闲时间（macOS）
#[cfg(target_os = "macos")]
fn get_system_idle_time() -> Result<u64, ()> {
    use std::process::Command;
    let output = Command::new("ioreg")
        .args(&["-c", "IOHIDSystem"])
        .output()
        .map_err(|_| ())?;

    let stdout = String::from_utf8_lossy(&output.stdout);
    for line in stdout.lines() {
        if line.contains("HIDIdleTime") {
            if let Some(ns) = line.split('=').nth(1) {
                let ns: u64 = ns.trim().parse().map_err(|_| ())?;
                return Ok(ns / 1_000_000_000); // 纳秒转秒
            }
        }
    }
    Err(())
}

#[cfg(not(target_os = "macos"))]
fn get_system_idle_time() -> Result<u64, ()> {
    Err(())
}
```

use tokio::sync::mpsc;

pub struct StorageManager {
    conn: Arc<Mutex<Connection>>,
    write_queue: mpsc::Sender<WriteTask>,  // 新增：写入队列
}

enum WriteTask {
    InsertCapture(NewCapture, tokio::sync::oneshot::Sender<Result<i64, StorageError>>),
    UpdateOcr(i64, String),
    // ... 其他写入操作
}

impl StorageManager {
    pub fn open(db_path: &Path) -> Result<Self, StorageError> {
        let conn = Connection::open(db_path)?;
        // ... 初始化代码 ...

        let conn = Arc::new(Mutex::new(conn));
        let (tx, mut rx) = mpsc::channel::<WriteTask>(1000);

        // 启动后台写入线程
        let conn_clone = conn.clone();
        tokio::spawn(async move {
            while let Some(task) = rx.recv().await {
                match task {
                    WriteTask::InsertCapture(new_capture, reply) => {
                        let result = {
                            let conn = conn_clone.lock().unwrap();
                            // 执行插入...
                        };
                        let _ = reply.send(result);
                    }
                    WriteTask::UpdateOcr(id, text) => {
                        let conn = conn_clone.lock().unwrap();
                        // 执行更新...
                    }
                }
            }
        });

        Ok(Self { conn, write_queue: tx })
    }

    pub async fn insert_capture_async(&self, new_capture: NewCapture) -> Result<i64, StorageError> {
        let (tx, rx) = tokio::sync::oneshot::channel();
        self.write_queue.send(WriteTask::InsertCapture(new_capture, tx))
            .await
            .map_err(|_| StorageError::Internal("写入队列已关闭".into()))?;
        rx.await.map_err(|_| StorageError::Internal("写入任务被取消".into()))?
    }
}

### 🛡️ 资源保护（推荐）

#### 修复 5: 添加资源监控

**新文件**: `core-engine/src/monitor.rs`

```rust
use sysinfo::{System, SystemExt, ProcessExt};

pub struct ResourceMonitor {
    system: System,
    max_memory_mb: u64,
    max_cpu_percent: f32,
}

impl ResourceMonitor {
    pub fn new() -> Self {
        Self {
            system: System::new_all(),
            max_memory_mb: 500,    // 最大 500MB
            max_cpu_percent: 50.0, // 最大 50% CPU
        }
    }

    pub fn should_throttle(&mut self) -> bool {
        self.system.refresh_all();

        let current_pid = sysinfo::get_current_pid().unwrap();
        if let Some(process) = self.system.process(current_pid) {
            let memory_mb = process.memory() / 1024 / 1024;
            let cpu_usage = process.cpu_usage();

            if memory_mb > self.max_memory_mb {
                warn!("内存使用过高: {} MB", memory_mb);
                return true;
            }

            if cpu_usage > self.max_cpu_percent {
                warn!("CPU 使用过高: {:.1}%", cpu_usage);
                return true;
            }
        }

        false
    }
}
```

**在 `main.rs` 中集成**:
```rust
let monitor = Arc::new(Mutex::new(ResourceMonitor::new()));

// 每 10 秒检查一次资源
let monitor_clone = monitor.clone();
let enabled = listener_config.enabled.clone();
tokio::spawn(async move {
    let mut interval = tokio::time::interval(Duration::from_secs(10));
    loop {
        interval.tick().await;
        let should_pause = monitor_clone.lock().unwrap().should_throttle();
        enabled.store(!should_pause, Ordering::Relaxed);
    }
});
```

## 总结

**核心问题**: 无节制的高频采集 + 阻塞式系统调用 + 缺少资源保护

**关键修复**:
1. ✅ 降低采集频率（30s → 60s）
2. ✅ 添加空闲检测（5 分钟无操作暂停）
3. ✅ osascript 异步化 + 2 秒超时
4. ✅ IPC 超时缩短（120s → 10s）
5. ✅ SQLite 写入队列化
6. ✅ 资源监控 + 自动限流

**预期效果**:
- CPU 占用从 80%+ 降至 < 10%
- 内存占用稳定在 200MB 以内
- 系统不再卡死
