### 2. ✅ 实现基于关键词的 RAG 查询

**功能描述**：
- 用户可以在问答界面输入任何问题
- 系统从知识库和采集记录中检索相关内容
- 返回检索结果和简单的答案摘要

**技术实现**：
- 使用 SQL LIKE 查询（因为 FTS5 没有配置中文分词器）
- 优先从知识库（knowledge_entries）检索
- 如果结果不足，从采集记录（captures）补充
- 返回最多 top_k 条相关记录

**查询逻辑**：
1. 从知识库检索：匹配 overview、details、summary 字段
2. 从采集记录检索：匹配 ocr_text、ax_text 字段
3. 每条记录限制 500 字符预览
4. 返回检索结果和上下文

**修改文件**：
- `core-engine/src/api/handlers/query.rs`

## API 接口

### POST /query

**请求参数**：
```json
{
  "query": "用户问题",
  "top_k": 5
}
```

**响应格式**：
```json
{
  "answer": "根据工作记录，我找到了 N 条相关信息：...",
  "contexts": [
    {
      "capture_id": 4860,
      "text": "相关内容...",
      "score": 1.0,
      "source": "knowledge"
    }
  ],
  "model": "fts5-retrieval"
}
```

**字段说明**：
- `answer`：简单的答案摘要
- `contexts`：检索到的相关上下文列表
- `model`：使用的检索模型（当前为 fts5-retrieval）

## 当前限制

### 1. 无 LLM 推理
- 当前版本只返回检索结果的简单摘要
- 没有调用本地 LLM 进行深度推理
- 提示用户："完整的 AI 推理功能正在开发中"

### 2. 中文分词问题
- FTS5 索引没有配置中文分词器
- 使用 SQL LIKE 进行关键词匹配
- 查询性能可能不如专业的全文检索

### 3. 查询精度
- 基于简单的关键词匹配
- 没有语义理解能力
- 可能返回不太相关的结果

## 技术架构

```
用户输入问题
    ↓
Desktop UI (RagPanel)
    ↓
HTTP POST /query
    ↓
Core Engine (query.rs)
    ↓
SQLite 数据库
    ├─ knowledge_entries (知识库)
    └─ captures (采集记录)
    ↓
SQL LIKE 关键词匹配
    ↓
返回检索结果
    ↓
组装答案摘要
    ↓
返回给前端显示
```
