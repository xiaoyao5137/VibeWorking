### 4. 数据库验证

#### 表结构
```sql
sqlite> .schema knowledge_entries
CREATE TABLE knowledge_entries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    capture_id INTEGER NOT NULL,
    summary TEXT NOT NULL,
    entities TEXT,
    category TEXT,
    importance INTEGER DEFAULT 3,
    user_verified BOOLEAN DEFAULT 0,
    user_edited BOOLEAN DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (capture_id) REFERENCES captures(id)
);
```

#### FTS5 索引
```sql
sqlite> .schema knowledge_fts
CREATE VIRTUAL TABLE knowledge_fts USING fts5(
    summary,
    entities,
    content='knowledge_entries',

);
```

## 发现的问题

### 1. ⚠️ 全文搜索中文分词问题

**问题描述**：
- 搜索 '产品' 找到 0 条结果（应该找到 1 条）
- 搜索 '会议' 找到 0 条结果（应该找到 2 条）
- 搜索 'OCR' 找到 1 条结果 ✅
- 搜索 'Python' 找到 1 条结果 ✅

**原因分析**：
SQLite FTS5 默认使用简单分词器，对中文支持不好。中文需要使用专门的分词器。

**解决方案**：
1. 使用 jieba 分词预处理中文文本
2. 或者使用 SQLite FTS5 的 unicode61 tokenizer
3. 或者使用外部搜索引擎（如 Elasticsearch）

### 2. ✅ 依赖问题已解决

**问题**：FastAPI 未安装导致模块导入失败

**解决**：修改 `__init__.py` 使 API 模块可选导入

```python
try:
    from .api import app as knowledge_api
    __all__ = ['KnowledgeExtractor', 'KnowledgeManager', 'knowledge_api']
except ImportError:
    __all__ = ['KnowledgeExtractor', 'KnowledgeManager']
```
