## 架构对比

### 原架构
```
请求 → Dispatcher → Worker（懒加载）→ 模型（常驻内存）
                                    ↓
                              内存占用: 8GB+
```

### 新架构
```
请求 → Dispatcher → 路由判断
                      ↓
        ┌─────────────┼─────────────┐
        │             │             │
    OCR（实时）   Embedding    RAG（按需）
    常驻 50MB     闲时队列     LLM 2.5GB
                      ↓
                 闲时引擎
                      ↓
              串行加载模型
              (1 小时 1 次)
                      ↓
            内存占用: 50MB - 3GB
```

---

## 核心改进

### 任务分类处理

| 任务类型 | 处理方式 | 响应时间 | 内存占用 |
|----------|----------|----------|----------|
| **OCR** | 实时处理 | < 1 秒 | 50MB（常驻） |
| **RAG** | 按需加载 | 首次 5 秒，后续 2 秒 | 2.5GB（2 分钟后卸载） |
| **Embedding** | 闲时队列 | 异步（1 小时内） | 650MB（闲时加载） |
| **VLM** | 闲时队列 | 异步（1 小时内） | 3GB（闲时加载） |
| **ASR** | 闲时队列 | 异步（1 小时内） | 80MB（闲时加载） |

### 内存占用优化

```
正常时间: 50MB ✅
用户提问: 2.5GB ✅
闲时计算: 3GB ✅
```

**对比原架构**: 从 8GB+ 降至 50MB - 3GB（节省 85%+）
