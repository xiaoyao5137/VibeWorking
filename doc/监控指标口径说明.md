# 监控指标口径说明

本文用于统一说明监控页中几个容易混淆的指标：`入索引`、`向量化率`、`知识化率`、`知识挂载率`。

## 1. 统计范围

以下指标默认都基于监控接口的时间范围参数（如 `1d` / `7d` / `30d`）计算。

在当前实现中，知识与向量化链路并不是对所有 `captures` 生效，而是只面向“可处理 captures”。

### 可处理 captures（eligible_count）

满足以下条件的 capture 会被计入“可处理 captures”：

- 在当前统计时间范围内
- `is_sensitive = 0`
- 且至少存在一种可处理文本：
  - `ocr_text` 非空，或
  - `ax_text` 非空

公式：

```text
eligible_count =
  count(captures in range
    where is_sensitive = 0
      and (ocr_text != '' or ax_text != ''))
```

说明：
- 敏感数据不会进入向量化/知识提炼链路，因此不应作为这些比率的分母。
- 没有 OCR/AX 文本的记录也无法进入这两条处理链路，因此同样不计入分母。

---

## 2. 入索引（vectorized_count）

### 含义

入索引表示：**已经成功写入 `vector_index` 的 capture 数量**。

它是一个数量值，不是百分比。

### 公式

```text
vectorized_count =
  count(distinct capture.id
    join vector_index on vector_index.capture_id = capture.id
    where capture ∈ eligible_count 的集合)
```

也可以理解为：

```text
入索引 = 已完成 embedding 且已写入向量索引的 capture 数
```

### 解释

一个 capture 只有在 embedding 成功、并且向量结果成功写入 `vector_index` 后，才算“入索引”。

---

## 3. 向量化率（vectorization_rate）

### 含义

向量化率表示：**可处理 captures 中，有多少已经完成向量化并成功入索引**。

### 公式

```text
vectorization_rate = vectorized_count / eligible_count
```

即：

```text
向量化率 = 已入索引数量 / 可处理 captures 数量
```

### 解释

这个指标反映的是 `capture -> embedding -> vector_index` 这条链路的完成度。

理想情况下，如果后台处理器持续稳定运行，向量化率会逐步接近 100%。

---

## 4. 知识化率（knowledge_generation_rate）

### 含义

知识化率表示：**可处理 captures 对应的数据中，已经生成出多少条 `knowledge_entries`**。

它关注的是“是否产出 knowledge 条目”。

### 公式

```text
knowledge_generated_count =
  count(knowledge_entries
    join captures on captures.id = knowledge_entries.capture_id
    where captures.ts in range)

knowledge_generation_rate = knowledge_generated_count / eligible_count
```

即：

```text
知识化率 = 已生成 knowledge_entries 数量 / 可处理 captures 数量
```

### 解释

知识提炼并不是一条 capture 对应一条 knowledge。通常是一组相关 captures 被合并后，生成一条 knowledge。

因此：
- 知识化率天然通常会低于向量化率
- 它更接近“知识沉淀效率”而不是“处理完成率”

---

## 5. 知识挂载率（knowledge_rate）

### 含义

知识挂载率表示：**可处理 captures 中，有多少已经成功回填了 `knowledge_id`**。

它关注的是“capture 是否已经挂到某条 knowledge 上”。

### 公式

```text
knowledge_linked_count =
  count(captures in range
    where knowledge_id is not null)

knowledge_rate = knowledge_linked_count / eligible_count
```

即：

```text
知识挂载率 = 已关联 knowledge_id 的 capture 数 / 可处理 captures 数量
```

### 解释

如果 5 条 captures 最终合并成 1 条 knowledge：
- 知识化率会增加 1
- 知识挂载率会增加 5 / eligible_count

因此知识挂载率通常会高于知识化率。

---

## 6. 三者关系

这几个指标关注的是不同层面：

### 向量化率
看的是：

```text
capture -> embedding -> vector_index
```

### 知识化率
看的是：

```text
一组 captures -> 1 条 knowledge_entries
```

### 知识挂载率
看的是：

```text
capture -> knowledge_id 已回填
```

通常可以预期：

```text
向量化率 >= 知识挂载率 >= 知识化率
```

但在短时间窗口内，也可能因为后台处理延迟、分组尚未成熟等原因出现暂时波动。

---

## 7. 示例

假设某个时间范围内：

- 可处理 captures = 100
- 入索引 captures = 90
- 生成 knowledge_entries = 12
- 已挂载 knowledge_id 的 captures = 40

则：

```text
入索引 = 90
向量化率 = 90 / 100 = 90%
知识化率 = 12 / 100 = 12%
知识挂载率 = 40 / 100 = 40%
```

这个例子说明：
- 向量化率衡量 embedding/索引链路
- 知识化率衡量知识条目产出
- 知识挂载率衡量 capture 被知识系统消化吸收的程度
