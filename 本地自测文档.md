## 一、环境准备

### 1.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS 12+ / Windows 10+ |
| CPU | Apple Silicon / Intel x64 |
| 内存 | 8GB+ (推荐 16GB) |
| 磁盘空间 | 20GB+ 可用空间 |

### 1.2 必需软件

#### 1.2.1 Rust 工具链

```bash
# 安装 Rust（如果未安装）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 验证安装
rustc --version  # 应显示 1.93.1 或更高
cargo --version
```

#### 1.2.2 Python 环境

```bash
# 验证 Python 版本（需要 3.11+）
python3 --version  # 应显示 3.11 或更高

# 创建虚拟环境
cd ai-sidecar
python3 -m venv .venv
source .venv/bin/activate  # macOS/Linux
# .venv\Scripts\activate   # Windows
```

#### 1.2.3 Node.js 环境（用于 UI）

```bash
# 验证 Node.js 版本（需要 18+）
node --version  # 应显示 v18 或更高
npm --version
```

#### 1.2.4 系统权限（macOS）

WorkBuddy 需要以下系统权限：

1. **辅助功能权限**（Accessibility）
   - 系统偏好设置 → 安全性与隐私 → 辅助功能
   - 添加终端应用（Terminal.app 或 iTerm2）

2. **屏幕录制权限**
   - 系统偏好设置 → 安全性与隐私 → 屏幕录制
   - 添加终端应用

## 三、启动服务

### 3.1 启动顺序

WorkBuddy 由 3 个独立进程组成，需按以下顺序启动：

```
1. AI Sidecar (Python)  ← 先启动，监听 IPC 连接
2. Core Engine (Rust)   ← 连接 Sidecar，启动采集和 API
3. Desktop UI (Tauri)   ← 连接 Core Engine API
```

### 3.2 启动 AI Sidecar

```bash
cd ai-sidecar
source .venv/bin/activate
python -m dispatcher
```

预期输出：
```
[2026-03-04 10:00:00] INFO - AI Sidecar 启动中...
[2026-03-04 10:00:01] INFO - IPC Server 监听: /tmp/workbuddy-sidecar.sock
[2026-03-04 10:00:02] INFO - 等待 Core Engine 连接...
```

**首次启动会下载模型**（约 2-5 分钟）：
- PaddleOCR 模型（~10MB）
- BGE-M3 Embedding 模型（~2GB）

### 3.3 启动 Core Engine

**新开一个终端窗口**：

```bash
cd core-engine
cargo run --release
```

预期输出：
```
[2026-03-04 10:00:05] INFO - WorkBuddy Core Engine 启动中...
[2026-03-04 10:00:06] INFO - 连接 AI Sidecar: /tmp/workbuddy-sidecar.sock
[2026-03-04 10:00:06] INFO - Sidecar 连接成功 (ping: 2ms)
[2026-03-04 10:00:07] INFO - 数据库初始化: ~/.workbuddy/workbuddy.db
[2026-03-04 10:00:07] INFO - 运行迁移: 001_init.sql, 002_fts.sql, 003_vector.sql
[2026-03-04 10:00:08] INFO - REST API 启动: http://localhost:7070
[2026-03-04 10:00:08] INFO - 采集引擎已启动（事件驱动模式）
```

### 3.4 启动 Desktop UI

**新开第三个终端窗口**：

```bash
cd desktop-ui

```

预期输出：

## 六、常见问题排查

### 6.1 AI Sidecar 无法启动

**症状**：
```
[ERROR] Failed to bind socket: Address already in use
```

**解决方案**：
```bash
# 检查端口占用
lsof -i :17071  # Windows
lsof /tmp/workbuddy-sidecar.sock  # macOS

# 杀死占用进程
kill -9 <PID>

# 或删除 socket 文件
rm /tmp/workbuddy-sidecar.sock
```

### 6.2 Core Engine 连接 Sidecar 失败

**症状**：
```
[ERROR] Failed to connect to Sidecar: Connection refused
```

**解决方案**：
1. 确认 AI Sidecar 已启动
2. 检查 socket 文件权限
```bash
ls -la /tmp/workbuddy-sidecar.sock
# 应显示 srwxr-xr-x
```

### 6.3 截图权限被拒绝（macOS）

**症状**：
```
[ERROR] Screenshot failed: Permission denied
```

**解决方案**：
1. 系统偏好设置 → 安全性与隐私 → 屏幕录制
2. 添加终端应用（Terminal.app）
3. 重启终端和 Core Engine

### 6.4 Accessibility 权限被拒绝（macOS）

**症状**：
```
[ERROR] AX API failed: kAXErrorAPIDisabled
```

**解决方案**：
1. 系统偏好设置 → 安全性与隐私 → 辅助功能
2. 添加终端应用
3. 重启终端和 Core Engine

### 6.5 OCR 识别失败

**症状**：
```
[ERROR] OCR failed: Model not loaded
```

**解决方案**：
```bash
cd ai-sidecar
source .venv/bin/activate

# 手动下载 PaddleOCR 模型
python -c "from paddleocr import PaddleOCR; PaddleOCR(use_angle_cls=True, lang='ch')"
```

### 6.6 数据库锁定错误

**症状**：
```
[ERROR] Database is locked
```

**解决方案**：
```bash
# 检查是否有多个 Core Engine 实例
ps aux | grep workbuddy-core

# 杀死多余进程
kill -9 <PID>

# 或删除 WAL 文件
rm ~/.workbuddy/workbuddy.db-wal
rm ~/.workbuddy/workbuddy.db-shm
```
