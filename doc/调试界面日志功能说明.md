## 使用方法

### 1. 启动调试界面
```bash
cd ai-sidecar
python start_debug_ui.py
```

### 2. 访问界面
```
http://localhost:8080
```

### 3. 测试任务提交

**测试 OCR**
1. 点击 "提交 OCR 任务" 按钮
2. 观察日志面板：
   ```
   10:30:15 提交任务: ocr-test-1234567890
   10:30:15 任务已提交（实时处理）
   ```

**测试 Embedding**
1. 点击 "提交 Embedding 任务" 按钮 5 次
2. 观察日志面板：
   ```
   10:30:00 提交任务: embed-test-1
   10:30:00 任务已加入闲时队列
   10:30:01 提交任务: embed-test-2
   10:30:01 任务已加入闲时队列
   ...
   ```
3. 等待闲时触发或点击 "手动触发闲时任务"
4. 观察日志更新：
   ```
   11:00:00 进入闲时模式
   11:00:00 开始加载模型
   11:00:03 模型加载完成
   11:00:15 处理完成 5 个任务
   ```

### 4. 清空日志
- 点击每个任务卡片右上角的 [清空] 按钮
- 该任务的日志将被清空

## 日志级别说明

| 级别 | 颜色 | 用途 |
|------|------|------|
| **info** | 蓝色 | 一般信息（提交任务、开始处理） |
| **success** | 绿色 | 成功操作（任务完成、模型加载） |
| **warning** | 橙色 | 警告信息（队列较满、资源不足） |
| **error** | 红色 | 错误信息（提交失败、处理异常） |

## 技术细节

### 日志存储
- 每个任务类型独立存储
- 内存中保留最近 100 条（任务日志）
- 内存中保留最近 200 条（系统日志）
- 超出自动删除最旧的

### 日志刷新
- 自动刷新：每 2 秒
- 手动刷新：提交任务后立即刷新
- WebSocket 推送：实时更新（未来可扩展）

### 性能优化
- 日志条数限制，避免内存溢出
- 滚动容器，避免 DOM 过大
- 批量更新，减少重绘
